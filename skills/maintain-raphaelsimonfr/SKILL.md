## Quick Reference

| Task | Location/Command |
|------|------------------|
| Create English post | `en/blog/posts/YYYY-MM-DD-slug/index.qmd` |
| Create French post | `fr/blog/posts/YYYY-MM-DD-slug/index.qmd` (same slug) |
| Preview site | `quarto preview` |
| Full render | `quarto render` (runs pre-render script automatically) |
| UI strings (TOC, search) | Automatic via `lang:` in `_metadata.yml` |
| Language switching | Automatic via `_includes/navbar-lang-switch.html` (JS) |

## Blog Post Frontmatter

```yaml
---
title: "Post Title"
description: "Brief description"
author: "RaphaГ«l Simon"
date: 2026-02-04
categories: [category1, category2]
image: thumbnail.jpg
translation: none    # none = original/manual | machine = LLM-translated
hreflang:            # For SEO (auto-generated for posts, manual for static pages)
  en: "/en/blog/posts/hello-world/"
  fr: "/fr/blog/posts/hello-world/"
---
```

| Field | Values | Purpose |
|-------|--------|---------|
| `translation` | `none` / `machine` | Controls machine-translation banner |
| `hreflang` | URL map per language | SEO alternate language links |

## Creating Content

### New Blog Post (Both Languages)

1. Create English version:
   ```
   en/blog/posts/2026-02-04-my-topic/index.qmd
   ```

2. Create French version (same slug):
   ```
   fr/blog/posts/2026-02-04-my-topic/index.qmd
   ```

3. Set `translation: none` in both if hand-written
4. Set `translation: machine` if one is LLM-translated

### Single-Language Post

Create only in one language directory. The other language's blog listing won't show it. Language switcher links to homepage for that language.

### Static Pages

`about.qmd`, `contact.qmd`, `thanks.qmd` exist in both `/en/` and `/fr/`. Edit each file directly вҖ” form labels, prose, everything is in the file itself.

## Multilingual Components

### Root Language Redirect

`index.html` at the site root detects the user's language preference:
1. Checks `localStorage` for `preferred-lang` (set on previous visits)
2. Falls back to `navigator.language` (French if starts with `fr`, else English)
3. Redirects to `/en/` or `/fr/`
4. Includes `<noscript>` fallback with manual links

### Navbar Language Switcher

Implemented in `_includes/navbar-lang-switch.html` (injected via `include-after-body` in `_quarto.yml`). This JavaScript runs on every page and:

1. Detects current language from URL pathname (`/en/` or `/fr/`)
2. Persists preference to `localStorage`
3. Rewrites all navbar links to match current language (e.g., `/en/about` вҶ’ `/fr/about`)
4. Translates navbar labels (About вҶ’ ГҖ propos)
5. Rewrites site brand link to current language homepage
6. Rewrites RSS feed link to current language
7. Adds **ENG | FRA** toggle to the right side of the navbar

**Note:** A `{{< lang-switch >}}` Quarto shortcode exists in `_extensions/multilang/lang-switch.lua` but is **not used** вҖ” the navbar JS handles all language switching.

### Translation Banner

Automatic via `translation-banner.lua` filter. When `translation: machine`:

> "Machine-translated вҖ” Original in: рҹҮ«рҹҮ·"

### hreflang SEO Tags

Automatic via `hreflang.lua` filter. Reads `hreflang` frontmatter and injects:
```html
<link rel="alternate" hreflang="en" href="..." />
<link rel="alternate" hreflang="fr" href="..." />
```

For blog posts: auto-generated by pre-render script.
For static pages: add manually to frontmatter once.

## Homepage Layout

Both `en/index.qmd` and `fr/index.qmd` use `page-layout: custom`, which hides Quarto's default title block (via CSS: `.page-layout-custom .quarto-title-block { display: none; }`).

The homepage uses a two-column grid layout:
- **Main column**: Recent posts listing (auto-populated from `blog/posts/`)
- **Sidebar column**: Highlights, featured posts, projects, reading links

On mobile (< 768px), columns stack vertically.

## Build Process

```yaml
# _quarto.yml
project:
  type: website
  pre-render:
    - scripts/build-manifest.py
```

Pre-render script:
1. Scans `/en/blog/posts/` and `/fr/blog/posts/`
2. Outputs `_data/translations-manifest.json`
3. Optionally auto-injects `hreflang` frontmatter

## UI String Translation

Quarto handles automatically via `lang:` setting:

```yaml
# en/_metadata.yml
lang: en

# fr/_metadata.yml
lang: fr
```

Built-in French translations for: TOC, search, listings, callouts, etc.

Override specific strings if needed:
```yaml
language:
  toc-title-document: "Sur cette page"
```

## RSS Feeds

Per-language:
- `/en/blog/index.xml`
- `/fr/blog/index.xml`

## Common Issues

| Problem | Solution |
|---------|----------|
| French UI strings in English | Check `lang: en` in `en/_metadata.yml` |
| Translation banner not showing | Verify `translation: machine` in frontmatter |
| Navbar links wrong language | Check `_includes/navbar-lang-switch.html` |
| hreflang missing | Add to frontmatter or check pre-render script |
| Homepage title showing | Ensure `page-layout: custom` in frontmatter |

## File Naming Rules

- **All filenames are English** (even in `/fr/`)
- **Blog dirs include date prefix**: `2026-02-02-hello-world/` not `hello-world/`
- **Blog slugs identical across languages**: `hello-world` not `bonjour-monde`
- **Separate files per language** for all content (no suffix convention like `.fr.qmd`)
